# 에셋 관리 정책

## 에셋의 정의

에셋(Asset)은 HTML 구조를 제외하고 페이지에 포함되는 리소스입니다.
- 이미지, 동영상, 링크 등 미디어 파일
- (향후) 은행 승인/결제 프로세스 등 외부 연동 리소스

에디터에서 파일을 선택할 때 사용하는 `ASSET_SELECTED` postMessage 이벤트가 이 개념을 반영합니다.

## 에셋 확장 계획 취소 배경 (2026-03-23)

기존 계획에서는 에셋에도 승인 프로세스를 적용하려 했으나, 아래 이유로 취소했습니다.

- **에셋 선택 UI 부재**: 에셋만 따로 선택·관리하는 화면을 새로 구현해야 함
- **미승인 에셋 차단 로직**: 컴포넌트 생성 시 미승인 에셋 유입을 막는 추가 로직 필요
- **만료 에셋 처리 정책 미결**: 에셋 기한 만료 시 기존 컴포넌트 처리 방식 미정

## 현재 운영 방향

| 항목 | 방침 |
|---|---|
| 에셋 저장 위치 | 로컬(`public/uploads/`) — DB 저장 없음 |
| 승인 처리 범위 | 페이지 단위만 승인·결재 프로세스 적용 |
| 컴포넌트 저장 방식 | HTML 형태 유지, 로컬 경로 에셋 참조 그대로 사용 |

## 페이지·에셋 동기화 임시 해결책

페이지(DB)와 에셋(로컬)이 분리되어 생기는 동기화 문제는 아래와 같이 보완합니다.

- **Github 활용**: 정적 파일(페이지 HTML, 에셋)을 Github에 올려 임시 관리
- **대체 이미지 처리**: 파일이나 페이지가 없는 경우 `alt` 속성으로 대체 아이콘·이미지 표시 (화면 깨짐 방지)

## API 경로 구분

| 경로 | 역할 |
|---|---|
| `/api/assets/*` | 에셋 목록 조회·등록·수정·삭제 (DB 메타데이터 관리) |
| `/api/builder/upload` | 에디터 내 파일 업로드 → `ASSET_UPLOAD_DIR`에 저장 |
| `/api/fal/*` | FAL AI를 통한 **AI 이미지 생성** → `ASSET_UPLOAD_DIR`에 결과 저장 |
