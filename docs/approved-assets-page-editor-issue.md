# CMS 페이지 편집 승인 이미지 사용 정책

## 개요

CMS 페이지 편집 시 로컬 PC에서 이미지를 직접 업로드해 사용하는 현재 구조를 변경한다.
페이지를 구성하는 이미지는 이미지 승인 관리에서 최종 승인된 이미지만 사용할 수 있도록 한다.

이미지 저장 위치 및 승인 후 이미지 제공 방식은 별도 작업자가 구현 중이므로, 본 이슈에서는 상세 구현 전 요구사항과 연동 지점을 정리한다.

## 배경

현재 `/cms/edit`에서는 일부 이미지 편집 기능에서 로컬 파일을 직접 업로드한다.

예시:

- `POST /cms/api/builder/upload`
- 업로드 즉시 `SPW_CMS_ASSET`에 이미지 메타데이터 저장
- 반환된 `ASSET_URL`을 페이지 HTML에 삽입
- 페이지는 해당 이미지 URL을 사용해 구성됨

문제는 이 흐름에서는 이미지가 승인되지 않아도 페이지에 포함될 수 있다는 점이다.

현재 admin에는 이미지 승인 관리 화면이 존재하며, 이미지 상태는 `SPW_CMS_ASSET.ASSET_STATE`로 관리된다.

```text
WORK -> PENDING -> APPROVED / REJECTED
```

앞으로는 `APPROVED` 상태의 이미지만 페이지에서 사용할 수 있어야 한다.

## 목표

- `/cms/edit`에서 로컬 이미지 직접 업로드를 통한 페이지 삽입을 제한한다.
- 페이지 편집자는 승인된 이미지 목록에서 이미지를 선택한다.
- 페이지 HTML에는 승인된 이미지의 URL 또는 식별자를 사용한다.
- 저장, 승인, 배포 과정에서 미승인 이미지 사용을 방지한다.

## 비목표

다음 작업은 본 이슈에서 직접 구현하지 않는다.

- 승인 이미지의 최종 저장 위치 설계
- 승인 전/승인 후 이미지 저장소 분리
- CDN 또는 public static 경로 구성
- 이미지 파일 이동 또는 복사 배치
- 이미지 서빙 API 최종 구현
- 기존 `ASSET_URL` 마이그레이션 정책

위 내용은 이미지 저장 위치 관련 별도 이슈에서 진행한다.

## 현재 관련 구조

### 이미지 메타데이터 테이블

```text
SPW_CMS_ASSET
```

주요 컬럼:

```text
ASSET_ID
ASSET_NAME
ASSET_PATH
ASSET_URL
ASSET_STATE
USE_YN
CREATE_USER_ID
CREATE_USER_NAME
```

승인 상태:

```text
WORK
PENDING
APPROVED
REJECTED
```

### 승인 이미지 목록 조회 가능 API

현재 `Springware-CMS`에는 에셋 목록 API가 존재한다.

```http
GET /cms/api/assets?assetState=APPROVED
```

이 API를 통해 승인된 이미지만 조회할 수 있다.

### 현재 로컬 업로드 경로

현재 일부 편집 컴포넌트는 아래 API를 직접 호출한다.

```http
POST /cms/api/builder/upload
```

예상 변경 방향:

```text
기존:
로컬 파일 선택 -> /api/builder/upload -> URL 반환 -> 페이지 삽입

변경:
승인 이미지 선택 -> /api/assets?assetState=APPROVED -> 이미지 선택 -> 페이지 삽입
```

## 요구사항

### 1. `/cms/edit`에서 승인 이미지 선택 기능 제공

페이지 편집 화면에서 이미지를 삽입할 때 로컬 파일 업로드 대신 승인된 이미지 목록을 선택할 수 있어야 한다.

예상 UI:

```text
이미지 선택 버튼
-> 승인 이미지 선택 모달
-> APPROVED 이미지 목록 표시
-> 이미지 선택
-> 현재 편집 중인 컴포넌트에 이미지 반영
```

조회 조건:

```http
GET /cms/api/assets?assetState=APPROVED
```

### 2. 로컬 이미지 직접 삽입 제한

`/cms/edit`에서 페이지 구성에 사용되는 이미지에 대해 로컬 파일 직접 업로드를 제한한다.

대상 예시:

- 배너 이미지
- 슬라이드 이미지
- 상품 카드 이미지
- 아이콘 이미지
- 팝업 배너 이미지
- 기타 페이지 HTML에 포함되는 이미지

현재 `POST /cms/api/builder/upload`를 호출하는 편집 컴포넌트는 승인 이미지 선택 방식으로 전환해야 한다.

### 3. 페이지 HTML에 이미지 식별 정보 보존

이미지 선택 시 단순 URL만 저장하는 것보다, 가능하면 `ASSET_ID`도 함께 남긴다.

예:

```html
<img src="{approvedAssetUrl}" data-asset-id="{assetId}" />
```

background image 형태인 경우:

```html
<div
  data-asset-id="{assetId}"
  style="background-image: url('{approvedAssetUrl}')"
></div>
```

이렇게 하면 저장, 승인, 배포 시 서버에서 승인 여부를 검증하기 쉽다.

### 4. 저장 시 서버 검증

`POST /cms/api/builder/save`에서 페이지 HTML에 포함된 이미지가 모두 승인된 이미지인지 검증한다.

검증 기준:

```text
SPW_CMS_ASSET.USE_YN = 'Y'
SPW_CMS_ASSET.ASSET_STATE = 'APPROVED'
```

검증 실패 시 저장을 차단한다.

응답 예:

```json
{
  "ok": false,
  "error": "승인되지 않은 이미지가 포함되어 있습니다."
}
```

### 5. 승인 요청 또는 배포 전 추가 검증

저장 시 검증 외에도 페이지 승인 요청 또는 배포 전 단계에서 한 번 더 검증한다.

이유:

- 기존 페이지 데이터에 미승인 이미지가 이미 포함되어 있을 수 있음
- DB 직접 수정 가능성
- 과거 저장 데이터와 신규 정책 간 호환성 문제

권장 검증 위치:

```text
페이지 저장 시
페이지 승인 요청 시
페이지 배포 직전
```

## 저장 위치 관련 별도 이슈와의 연계

이미지 저장 위치 관련 이슈에서 결정해야 할 내용:

```text
승인 전 이미지 저장 위치
승인 후 이미지 저장 위치
승인 후 이미지 URL 생성 방식
이미지 공개 접근 경로
이미지 파일 이동/복사 정책
기존 ASSET_URL 마이그레이션 정책
```

본 이슈에서는 저장 위치가 확정되면 해당 URL 또는 이미지 조회 API를 사용한다.

예상 연동 방식 중 하나:

```json
{
  "assetId": "...",
  "assetName": "...",
  "url": "승인 후 사용 가능한 이미지 URL",
  "assetState": "APPROVED"
}
```

`/cms/edit`은 이 `url`과 `assetId`를 사용해 페이지 HTML을 구성한다.

## 고려사항

### 기존 미승인 이미지가 포함된 페이지

정책 적용 전 생성된 페이지에는 미승인 이미지가 포함되어 있을 수 있다.

처리 방안 필요:

```text
1. 저장 시 차단
2. 승인 요청 시 차단
3. 관리자에게 미승인 이미지 목록 표시
4. 승인 이미지로 교체 유도
```

### 외부 URL 이미지

사용자가 임의 외부 URL 이미지를 넣을 수 있는 컴포넌트가 있다면 정책을 정해야 한다.

권장:

```text
페이지 구성 이미지는 SPW_CMS_ASSET 기반 승인 이미지로 제한
외부 이미지 URL 직접 입력은 금지 또는 별도 예외 정책 필요
```

### 성능

승인 이미지 목록이 많아질 수 있으므로 페이지네이션, 검색, 카테고리 필터가 필요하다.

필요 필터 예:

```text
업무 카테고리
파일명
등록자
등록일
```

## 완료 조건

- `/cms/edit`에서 페이지 구성용 이미지는 승인된 이미지 목록에서만 선택할 수 있다.
- 로컬 파일 업로드로 이미지를 즉시 페이지에 삽입할 수 없다.
- 선택된 이미지에는 `ASSET_ID` 추적 정보가 남는다.
- 페이지 저장 시 포함된 이미지가 모두 `APPROVED` 상태인지 검증한다.
- 승인되지 않은 이미지가 포함된 경우 저장 또는 승인 요청이 차단된다.
- 이미지 저장 위치 관련 별도 이슈에서 제공하는 승인 이미지 URL 정책과 연동된다.
