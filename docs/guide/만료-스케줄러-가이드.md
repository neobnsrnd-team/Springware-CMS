# 페이지 만료 스케줄러 가이드

## 개요

`EXPIRED_DATE`가 경과된 승인 페이지를 자동으로 비공개 처리하는 만료 관리 시스템입니다.

---

## 만료 처리 흐름

```
스케줄러 호출 (수동 또는 cron)
→ POST /api/scheduler/expire
→ 1. 만료 페이지 조회 (EXPIRED_DATE < 오늘, IS_PUBLIC='Y', USE_YN='Y')
→ 2. 각 페이지별:
   a. 원본 HTML → public/uploads/pages/backup/ 으로 이동 (백업)
   b. public/system/page-expired.html → 원본 위치에 복사 (만료 안내 치환)
   c. DB: IS_PUBLIC='N', FILE_PATH_BACK 기록
→ 3. 처리 결과 반환
```

---

## 사용 방법

### 1. 환경변수 설정

`.env`에 스케줄러 인증 토큰을 설정합니다:

```
SCHEDULER_SECRET=임의의-비밀-문자열
```

### 2. 수동 실행

dev 서버가 실행 중인 상태에서 다른 터미널에서:

```bash
curl -X POST http://localhost:3000/api/scheduler/expire \
  -H "x-scheduler-token: 임의의-비밀-문자열"
```

### 3. OS cron 자동 실행 (운영 환경)

매일 자정에 실행하는 예시:

```bash
# crontab -e
0 0 * * * curl -s -X POST http://localhost:3000/api/scheduler/expire -H "x-scheduler-token: 임의의-비밀-문자열" >> /var/log/cms-expire.log 2>&1
```

### 4. 응답 형식

```json
{
  "ok": true,
  "processed": 3,
  "failed": [
    { "pageId": "page-001", "error": "파일 이동 실패" }
  ]
}
```

- `processed`: 정상 처리된 페이지 수
- `failed`: 실패한 페이지 목록 (부분 실패 허용 — 하나 실패해도 나머지 계속 처리)

## IS_PUBLIC 긴급 차단

관리자가 특정 페이지를 즉시 비공개/공개로 전환합니다:

```bash
# 비공개 전환
curl -X PATCH http://localhost:3000/api/builder/pages/{pageId}/set-public \
  -H "Content-Type: application/json" \
  -H "Cookie: cms-user=admin" \
  -d '{"isPublic": "N"}'

# 공개 복원
curl -X PATCH http://localhost:3000/api/builder/pages/{pageId}/set-public \
  -H "Content-Type: application/json" \
  -H "Cookie: cms-user=admin" \
  -d '{"isPublic": "Y"}'
```

## 상태 구분

| 상태 | USE_YN | IS_PUBLIC | EXPIRED_DATE | 설명 |
|---|---|---|---|---|
| 정상 공개 | Y | Y | null/미래 | 대고객 노출 중 |
| 만료 | Y | N | 과거 | 만료일 경과로 비공개 |
| 긴급 차단 | Y | N | null/미래 | 관리자 수동 비공개 |
| 삭제 (소프트) | N | - | - | 승인 이력 있는 삭제 |
| 삭제 (하드) | - | - | - | 레코드 물리 삭제 |

---

## 파일 경로

| 용도 | 경로 |
|---|---|
| 원본 HTML | `public/uploads/pages/{pageId}.html` |
| 백업 HTML | `public/uploads/pages/backup/{pageId}.html` |
| 만료 안내 페이지 | `public/system/page-expired.html` |
