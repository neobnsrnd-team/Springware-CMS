# 새 컴포넌트 구현 가이드

ContentBuilder(Innova)의 네이티브 기능을 최대한 활용하는 것을 원칙으로 합니다.
새 컴포넌트는 **순수 HTML** 방식을 기본으로 하며, JS 런타임 의존이 불가피한 경우에만 플러그인 방식을 선택합니다.

---

## 개발 경로 선택

```
새 컴포넌트 구현
        │
        ▼
JS 런타임 의존이 필요한가?
(실시간 fetch, 지도 렌더링, 복잡한 계산 등)
        │
  ┌─────┴─────┐
 아니오       예
  │            │
  ▼            ▼
순수 HTML    플러그인
 (기본)     (예외적 선택)
```

**순수 HTML**: 텍스트·이미지·링크·레이아웃으로 표현 가능한 컴포넌트. ContentBuilder 인라인 편집(더블클릭) 활용 가능.

**플러그인 유지 대상** (현재): `exchange-board`, `loan-calculator`

---

## Path A — 순수 HTML 컴포넌트 (기본)

### 체크리스트

```
□ 1. 순수 HTML 작성 (3개 variant)
□ 2. scripts/migrate-<name>-to-html.ts 작성
□ 3. 마이그레이션 스크립트 실행 (DB INSERT or UPDATE)
□ 4. SVG 썸네일 생성
□ 5. (필요 시) 커스텀 편집 기능 추가
```

---

### 1. 순수 HTML 작성

**루트 요소 필수 속성 2개**:

| 속성 | 역할 |
|------|------|
| `data-component-id="<name>-<viewmode>"` | DB `COMPONENT_ID`와 1:1 매핑. 페이지-컴포넌트 추적에 사용 |
| `data-spw-block` | `handleInsertComponent`가 감지 → 컬럼에 `spw-finance-col` 클래스 주입 (캔버스 전체 너비) |

**스타일 규칙**:
- 모든 CSS 클래스 스타일 → **인라인 `style=""`로 작성**
- `@container`, `@media`, CSS 클래스 사용 불가 (ContentBuilder가 외부 CSS를 격리함)
- `overflow: hidden` 사용 금지 (ContentBuilder 툴바가 잘림)
- 폰트: `-apple-system, BlinkMacSystemFont, 'Malgun Gothic', 'Apple SD Gothic Neo', sans-serif`

**3개 variant 차이**:

| variant | 특징 |
|---------|------|
| `mobile` | 기준 너비 390px, 작은 패딩·폰트 |
| `web` | 넓은 패딩, `max-width` + `margin: 0 auto` |
| `responsive` | `width: 100%; box-sizing: border-box` |

**작성 예시** (`app-header` 패턴):

```html
<!-- mobile variant -->
<div data-component-id="my-block-mobile" data-spw-block
     style="font-family:-apple-system,BlinkMacSystemFont,'Malgun Gothic',sans-serif;background:#ffffff;border-bottom:2.5px solid #0046A4;">
    <div style="display:flex;align-items:center;justify-content:center;height:54px;padding:0 14px;">
        <a href="#" style="display:flex;align-items:center;gap:8px;text-decoration:none;">
            <span style="font-size:17px;font-weight:800;color:#1A1A2E;">컴포넌트 제목</span>
        </a>
    </div>
</div>
```

> `<a>` 태그를 반드시 포함할 것. ContentBuilder의 `#divLinkTool`을 여는 진입점이 되며,
> 커스텀 편집 버튼을 추가하려면 이 링크 태그가 필요합니다.

### 2. 마이그레이션 스크립트 작성

**파일 위치**: `scripts/migrate-<name>-to-html.ts`

```ts
import 'dotenv/config';
import { getComponentById, updateComponent, createComponent } from '../src/db/repository/component.repository';
import { closePool } from '../src/db/connection';

const FONT_FAMILY = "-apple-system,BlinkMacSystemFont,'Malgun Gothic','Apple SD Gothic Neo',sans-serif";

const MOBILE_HTML = `<div data-component-id="my-block-mobile" data-spw-block style="...">
    <!-- HTML -->
</div>`;

const WEB_HTML = `<div data-component-id="my-block-web" data-spw-block style="...">
    <!-- HTML -->
</div>`;

const RESPONSIVE_HTML = `<div data-component-id="my-block-responsive" data-spw-block style="...">
    <!-- HTML -->
</div>`;

const VARIANTS = [
    { id: 'my-block-mobile',     html: MOBILE_HTML,     viewMode: 'mobile',     label: '내 컴포넌트', description: '설명' },
    { id: 'my-block-web',        html: WEB_HTML,        viewMode: 'web',        label: '내 컴포넌트', description: '설명' },
    { id: 'my-block-responsive', html: RESPONSIVE_HTML, viewMode: 'responsive', label: '내 컴포넌트', description: '설명' },
];

async function main() {
    for (const variant of VARIANTS) {
        const existing = await getComponentById(variant.id);

        if (existing) {
            // 이미 존재 → HTML만 업데이트 (COMPONENT_TYPE, VIEW_MODE 보존 필수)
            await updateComponent({
                componentId:        variant.id,
                componentType:      existing.COMPONENT_TYPE,
                viewMode:           existing.VIEW_MODE,
                componentThumbnail: existing.COMPONENT_THUMBNAIL ?? undefined,
                data: {
                    ...(existing.DATA ?? {}) as Record<string, unknown>,
                    html: variant.html,
                },
                lastModifierId: 'system',
            });
            console.log(`✅ ${variant.id} — UPDATE 완료`);
        } else {
            // 신규 컴포넌트 → INSERT
            await createComponent({
                componentId:      variant.id,
                componentType:    'finance',
                viewMode:         variant.viewMode as 'mobile' | 'web' | 'responsive',
                componentThumbnail: `/assets/minimalist-blocks/preview/ibk-my-block.svg`,
                data: {
                    id:          variant.id.replace(`-${variant.viewMode}`, ''),
                    label:       variant.label,
                    description: variant.description,
                    preview:     `/assets/minimalist-blocks/preview/ibk-my-block.svg`,
                    html:        variant.html,
                    viewMode:    variant.viewMode,
                },
                createUserId:   'system',
                createUserName: '시스템',
            });
            console.log(`✅ ${variant.id} — INSERT 완료`);
        }
    }
    await closePool();
}

main().catch((err) => { console.error('실패:', err); process.exit(1); });
```

> **주의**: `updateComponent` 시 `componentType`과 `viewMode`는 반드시 기존값 전달.
> 생략하거나 직접 입력하면 `null`로 덮어써집니다.

---

### 3. 마이그레이션 스크립트 실행

```bash
npx tsx scripts/migrate-<name>-to-html.ts
```

실행 후 에디터에서 해당 컴포넌트를 패널에서 **새로 드래그** → 더블클릭으로 인라인 편집이 가능한지 확인합니다.

> 이미 저장된 페이지의 기존 블록은 마이그레이션 대상이 아닙니다.
> 해당 블록은 삭제 후 새로 추가해야 합니다.

### 4. SVG 썸네일

**파일 위치**: `public/assets/minimalist-blocks/preview/ibk-<name>.svg`

```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 390 <높이>" width="390" height="<높이>">
  <rect width="390" height="<높이>" fill="#fff"/>
  <rect x="16" y="16" width="80" height="20" rx="4" fill="#0046A4"/>
  <text x="16" y="60" font-size="14" fill="#374151">컴포넌트 이름</text>
</svg>
```

권장 높이:
- 헤더/푸터: 56–160px
- 아이콘 그리드: 200–320px
- 카드/배너: 200–240px

---

### 5. 커스텀 편집 기능 (필요 시)

ContentBuilder 인라인 편집(더블클릭)으로 처리할 수 없는 구조화 콘텐츠(SVG 아이콘, iframe, select 등)가 있는 경우에만 추가합니다.

→ **[커스텀-편집-기능-개발-가이드.md](./커스텀-편집-기능-개발-가이드.md)** 참고

완성 예시:
- `product-menu` → `src/components/edit/ProductMenuIconEditor.tsx` (SVG 아이콘 편집)
- `media-video` → `src/components/edit/MediaVideoEditor.tsx` (YouTube URL 변경)
- `auth-center` → `src/components/edit/AuthCenterIconEditor.tsx` (아이콘+색상 편집)
- `site-footer` → `src/components/edit/SiteFooterSelectEditor.tsx` (`<select>` 캡처 클릭 패턴)
- `app-header` → `src/components/edit/AppHeaderBorderEditor.tsx` (구분선 색상·굵기 편집)

## Path B — 플러그인 컴포넌트 (JS 의존)

JS 런타임(실시간 fetch, 지도, 복잡한 계산 등)이 반드시 필요한 경우에만 선택합니다.

### 체크리스트

```
□ 1. public/assets/plugins/<name>/style.css   — 컴포넌트 CSS
□ 2. public/assets/plugins/<name>/index.js    — 컴포넌트 JS (settings · editor · mount)
□ 3. SVG 썸네일
□ 4. DB에 컴포넌트 INSERT (마이그레이션 스크립트)
□ 5. EditClient.tsx — Runtime 플러그인 등록
□ 6. ViewClient.tsx — Runtime 플러그인 등록 (미리보기용)
```

### 5 & 6. Runtime 플러그인 등록

**`src/components/edit/EditClient.tsx`** 및 **`src/app/view/ViewClient.tsx`** 의 `plugins` 객체에 동일하게 추가:

```ts
'<name>': {
    url: basePath + '/assets/plugins/<name>/index.js',
    css: basePath + '/assets/plugins/<name>/style.css',
},
```

### style.css 핵심 규칙

```css
/* data-cb-type 선택자로 스코프 한정 */
[data-cb-type="<name>"] {
    container-type: inline-size; /* Container Query 기반 반응형 — 필수 */
    font-family: -apple-system, BlinkMacSystemFont, 'Malgun Gothic', 'Apple SD Gothic Neo', sans-serif;
    /* overflow: hidden 금지 — ContentBuilder 툴바가 잘림 */
}

/* 모든 클래스는 반드시 부모 선택자 앞에 붙임 */
[data-cb-type="<name>"] .my-class { ... }

/* viewport가 아닌 컨테이너 기준 반응형 */
@container (max-width: 360px) { ... }

/* 다크 모드 */
[data-cb-skin="dark"] [data-cb-type="<name>"] { background: #1F2937; }
```

### index.js 구조

```js
export default {
    name: '<name>',           // data-cb-type 값과 동일
    displayName: '표시 이름',
    version: '1.0.0',

    settings: {
        accentColor: { type: 'color', label: '포인트 색상', default: '#0046A4' },
    },

    editor: {
        openContentEditor: function(element, builder, onChange) {
            const container = document.createElement('div');
            // element: 캔버스의 [data-cb-type] 루트 DOM
            // onChange: 변경 시 반드시 호출
            return container; // 반드시 DOM 요소를 반환
        }
    },

    mount: function(element, options) {
        // options에 settings 키-값이 들어옴
        return {}; // instance 객체
    },

    unmount: function(element, instance) {
        // mount에서 등록한 리스너 정리
    }
};
```

### DB INSERT

플러그인 컴포넌트도 동일하게 마이그레이션 스크립트로 DB에 INSERT합니다.
다만 HTML에 `data-cb-type="<name>"` 속성을 사용하고, `data-component-id`·`data-spw-block`은 불필요합니다.

## 구현 순서 요약

### 순수 HTML (Path A)

```
1. HTML 작성 (3 variant) → 2. 마이그레이션 스크립트 작성
→ 3. npx tsx scripts/migrate-<name>-to-html.ts
→ 4. SVG 썸네일 생성
→ 5. (필요 시) 커스텀 편집 컴포넌트 추가
→ npm run dev → /edit 에서 확인
```

### 플러그인 (Path B)

```
1. style.css 작성 → 2. index.js 작성 → 3. SVG 썸네일 생성
→ 4. 마이그레이션 스크립트로 DB INSERT
→ 5. EditClient.tsx + ViewClient.tsx 플러그인 등록
→ npm run dev → /edit 에서 확인
```
