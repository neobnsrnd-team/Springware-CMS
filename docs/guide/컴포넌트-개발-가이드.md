# 새 컴포넌트 구현 가이드

새 컴포넌트를 추가할 때 수정해야 하는 파일 목록과 각 파일에 작성할 내용을 순서대로 정리합니다.

---

## 전체 체크리스트

```
□ 1. public/assets/plugins/<name>/style.css       — 컴포넌트 CSS
□ 2. public/assets/plugins/<name>/index.js        — 컴포넌트 JS (settings · editor · mount)
□ 3. public/assets/minimalist-blocks/preview/ibk-<name>.svg  — 패널 썸네일
□ 4. src/app/edit/finance-component-data.ts       — 우측 패널 데이터 추가
□ 5. src/app/edit/EditClient.tsx                  — Runtime 플러그인 등록
□ 6. src/app/view/ViewClient.tsx                  — Runtime 플러그인 등록 (미리보기용)
```

---

## 1. `style.css` — 컴포넌트 CSS

**파일 위치:** `public/assets/plugins/<name>/style.css`

### 필수 규칙

```css
/* 반드시 data-cb-type 속성 선택자로 스코프를 한정 */
[data-cb-type="<name>"] {
    container-type: inline-size;   /* Container Query 기반 반응형 — 필수 */
    font-family: -apple-system, BlinkMacSystemFont, 'Malgun Gothic', 'Apple SD Gothic Neo', sans-serif;
    background: #ffffff;
    position: relative;
    /* overflow: hidden 금지 — ContentBuilder 툴바가 잘림 */
}

/* 모든 클래스는 반드시 부모 선택자를 앞에 붙임 */
[data-cb-type="<name>"] .my-class { ... }

/* 반응형: viewport가 아닌 컨테이너 기준으로 작성 */
@container (max-width: 360px) {
    .my-class { font-size: 14px; }
}

/* 다크 모드 */
[data-cb-skin="dark"] [data-cb-type="<name>"] { background: #1F2937; }
```

### 핵심 제약
- `overflow: hidden` 절대 사용 금지 (행도구 툴바가 `.column` 바깥에 렌더링됨)
- `@media` 대신 `@container` 사용 (390px 캔버스에서 정확히 동작)
- 클래스명은 컴포넌트 고유 접두어를 붙여 충돌 방지 (예: `qb-`, `tn-`, `pb-`)

## 2. `index.js` — 컴포넌트 JS

**파일 위치:** `public/assets/plugins/<name>/index.js`

### 전체 구조

```js
export default {
    name: '<name>',           // data-cb-type 값과 동일
    displayName: '표시 이름', // ContentBuilder 내부 표시용
    version: '1.0.0',

    // ── ① settings: 우측 패널 톱니바퀴(⚙) 클릭 시 나타나는 설정 옵션
    settings: {
        accentColor: {
            type: 'color',    // 'color' | 'boolean' | 'number' | 'text'
            label: '포인트 색상',
            default: '#0046A4'
        },
        showHeader: {
            type: 'boolean',
            label: '헤더 표시',
            default: true
        },
        columns: {
            type: 'number',
            label: '열 수',
            default: 4,
            min: 2,
            max: 5
        }
    },

    // ── ② editor: 우측 패널 연필(✏) 클릭 시 나타나는 콘텐츠 편집 UI
    editor: {
        openContentEditor: function(element, builder, onChange) {
            const container = document.createElement('div');

            // DOM을 직접 조작해 입력 UI를 만들고 반환
            // element: 캔버스의 [data-cb-type] 루트 DOM
            // onChange: 변경 시 반드시 호출 (ContentBuilder에 변경 알림)

            const input = document.createElement('input');
            input.type = 'text';
            input.value = element.querySelector('.my-title')?.textContent || '';
            input.addEventListener('input', () => {
                element.querySelector('.my-title').textContent = input.value;
                onChange?.();
            });
            container.appendChild(input);

            return container; // 반드시 DOM 요소를 반환
        }
    },

    // ── ③ mount: 컴포넌트가 캔버스/뷰에 삽입될 때 실행
    mount: function(element, options) {
        // options에는 settings에서 정의한 키-값이 들어옴
        const accent = options.accentColor || '#0046A4';

        // CSS 변수 적용
        element.style.setProperty('--my-accent', accent);

        // 에디터 내 링크 클릭 차단 (페이지 이동 방지)
        element.querySelectorAll('a').forEach(a => {
            a.addEventListener('click', e => e.preventDefault());
        });

        return {}; // instance 객체 (unmount에서 사용할 경우)
    },

    unmount: function(element, instance) {
        // mount에서 등록한 전역 리스너 등 정리
    }
};
```

### editor.openContentEditor 패턴 (항목 목록 편집)

퀵뱅킹처럼 항목이 여러 개인 경우 아래 패턴을 사용합니다.

```js
openContentEditor: function(element, builder, onChange) {
    const container = document.createElement('div');

    // 공용 스타일 상수
    const S = {
        row:   'display:flex;align-items:center;gap:8px;margin-bottom:8px;',
        label: 'font-size:12px;color:#6b7280;width:76px;flex-shrink:0;',
        input: 'flex:1;padding:5px 9px;border:1px solid #e5e7eb;border-radius:5px;font-size:13px;outline:none;',
    };

    // 항목 리스트 렌더 (추가/삭제/순서변경 후 재호출)
    const listWrap = document.createElement('div');
    container.appendChild(listWrap);

    const renderList = () => {
        listWrap.innerHTML = '';
        element.querySelectorAll('.my-item').forEach((item, idx, all) => {
            // 카드: 헤더(▲▼ 이름 ✕) + 바디(접힘/펼침 편집 폼)
            const card = document.createElement('div');
            // ... (quick-banking/index.js 참조)
            listWrap.appendChild(card);
        });
    };
    renderList();

    // 항목 추가 버튼
    const addBtn = document.createElement('button');
    addBtn.textContent = '+ 항목 추가';
    addBtn.onclick = () => {
        const newItem = document.createElement('a');
        newItem.className = 'my-item';
        newItem.href = '#';
        newItem.innerHTML = `<span class="my-label">새 항목</span>`;
        element.querySelector('.my-list').appendChild(newItem);
        onChange?.();
        renderList();
    };
    container.appendChild(addBtn);

    return container;
}
```

## 3. SVG 썸네일

**파일 위치:** `public/assets/minimalist-blocks/preview/ibk-<name>.svg`

```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 390 <높이>" width="390" height="<높이>">
  <!-- 컴포넌트의 대략적인 모양을 단순화해서 표현 -->
  <!-- 세부 구현 없이 색상 블록/텍스트로만 구성해도 충분 -->
  <rect width="390" height="<높이>" fill="#fff"/>
  <rect x="16" y="16" width="80" height="20" rx="4" fill="#0046A4"/>
  <text x="16" y="60" font-size="14" fill="#374151">컴포넌트 이름</text>
</svg>
```

권장 높이 기준:
- 헤더류: 56px
- 아이콘 그리드: 200~320px
- 카드/배너: 200~240px
- 푸터: 160~200px

## 4. `finance-component-data.ts` — 패널 데이터

**파일 위치:** `src/app/edit/finance-component-data.ts`

`FINANCE_COMPONENTS` 배열 끝에 항목 추가:

```ts
{
    id: '<name>',                          // data-cb-type 값과 동일
    label: '컴포넌트 표시 이름',
    description: '한 줄 설명',
    preview: '/assets/minimalist-blocks/preview/ibk-<name>.svg',
    html: `
<div data-cb-type="<name>">
    <!-- 초기 HTML 구조 -->
    <!-- 클래스명은 반드시 고유 접두어 사용 -->
</div>
`,
},
```

> `html` 필드는 ContentBuilder가 `<div class="row"><div class="column">…</div></div>`로 자동 감싸므로
> 루트 `<div data-cb-type>` 만 작성합니다.

## 5 & 6. Runtime 플러그인 등록

두 파일에 동일하게 추가합니다.

**EditClient.tsx** (`src/app/edit/EditClient.tsx`) — 에디터용
**ViewClient.tsx** (`src/app/view/ViewClient.tsx`) — 미리보기용

```ts
// plugins 객체 안에 추가 (IBK 금융 컴포넌트 섹션)
'<name>': {
    url: basePath + '/assets/plugins/<name>/index.js',
    css: basePath + '/assets/plugins/<name>/style.css'
},
```

두 파일 모두 `plugins: { ... }` 블록에 동일하게 추가해야 미리보기(`/view`)에서도 정상 렌더링됩니다.

## 자주 쓰는 CSS 패턴

```css
/* 카드 기본 */
[data-cb-type="<name>"] .my-card {
    background: #fff;
    border-radius: 16px;
    box-shadow: 0 2px 16px rgba(0,0,0,0.07);

    -webkit-tap-highlight-color: transparent;
    cursor: pointer;
    transition: background 0.15s;
}
[data-cb-type="<name>"] .my-btn:active {
    transform: scale(0.96);
}

/* 스크롤 트랙 (가로 스와이프) */
[data-cb-type="<name>"] .my-track {
    display: flex;
    overflow-x: auto;
    scroll-snap-type: x mandatory;
    -webkit-overflow-scrolling: touch;
    scrollbar-width: none;
}
[data-cb-type="<name>"] .my-track::-webkit-scrollbar { display: none; }

/* 스와이프 슬라이드 */
[data-cb-type="<name>"] .my-slide {
    scroll-snap-align: start;
    flex: 0 0 100%;
    min-width: 100%;
}
```
