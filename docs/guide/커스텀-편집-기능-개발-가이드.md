# 커스텀 편집 기능 개발 가이드

ContentBuilder가 제공하지 않는 도메인 특화 편집 기능을 추가하는 패턴을 정리합니다.

---

## 핵심 패턴: `#divLinkTool` 버튼 주입

ContentBuilder는 캔버스 안의 `<a>` 태그를 클릭하면 `#divLinkTool` 툴바를 DOM에 동적으로 표시한다.
여기에 커스텀 버튼을 주입하고, 활성 요소 위치에 따라 버튼 가시성을 제어한다.

**전제 조건: 편집 대상 블록 안에 `<a>` 태그가 반드시 있어야 한다.**

```
<a> 클릭
  → ContentBuilder가 #divLinkTool을 DOM에 추가
  → MutationObserver 감지 → 커스텀 버튼 주입 (최초 1회)
  → .icon-active / .elm-active 클래스 변화 감지 → 버튼 가시성 제어
  → 버튼 클릭 → React 모달 열기
```

---

## 구현 체크리스트

### 1. 블록 HTML에 `<a>` 태그 배치

편집 진입점이 될 `<a href="#">` 태그를 블록 안에 포함시킨다.
기존 컴포넌트 HTML을 변경한 경우 마이그레이션 스크립트 실행 + 이미 저장된 페이지 HTML도 직접 수정한다.

> `public/uploads/pages/*.html` 파일과 DB 양쪽 모두 업데이트해야 한다.

### 2. 편집 패널 컴포넌트 작성

`src/components/edit/XxxEditor.tsx`로 만든다. `Modal` 공통 컴포넌트 대신 **자체 패널 구조**를 사용한다.

```tsx
// src/components/edit/MyBlockEditor.tsx
export default function MyBlockEditor({ blockEl, onClose }: { blockEl: HTMLElement; onClose: () => void }) {
    const handleApply = () => {
        // blockEl.querySelector(...)로 DOM 직접 수정
        // ContentBuilder는 다음 Save 시 현재 DOM을 읽어 저장 → 별도 동기화 불필요
        onClose();
    };

    return (
        <>
            {/* 오버레이 */}
            <div onClick={onClose} style={{ position: 'fixed', inset: 0, zIndex: 99998, background: 'rgba(0,0,0,0.35)' }} />
            {/* 패널 */}
            <div style={{ position: 'fixed', left: '50%', top: '50%', transform: 'translate(-50%,-50%)', width: 360, zIndex: 99999, background: '#fff', border: '1px solid #e5e7eb', borderRadius: 12, boxShadow: '0 8px 32px rgba(0,0,0,0.18)' }}>
                {/* 헤더 */}
                <div style={{ padding: '12px 14px', borderBottom: '1px solid #f3f4f6', background: '#fafafa', borderRadius: '12px 12px 0 0', display: 'flex', justifyContent: 'space-between', alignItems: 'center' }}>
                    <span style={{ fontWeight: 700, color: '#111827' }}>편집 제목</span>
                    <button onClick={onClose} style={{ border: 'none', background: 'none', cursor: 'pointer', color: '#6b7280', fontSize: 18, lineHeight: 1 }}>×</button>
                </div>
                {/* 본문 — padding: '14px 14px 10px' */}
                {/* 버튼 영역 — padding: '10px 14px 14px', borderTop: '1px solid #f3f4f6' */}
            </div>
        </>
    );
}
```

### 3. EditClient.tsx에 버튼 추가

`injectCustomButtonsToLinkTool` 함수에 버튼 블록 추가:

```ts
const SPW_MY_BTN_CLASS = 'spw-my-btn';

if (!linkTool.querySelector(`.${SPW_MY_BTN_CLASS}`)) {
    const btn = document.createElement('button');
    btn.className = SPW_MY_BTN_CLASS;
    btn.title = '편집';
    btn.style.cssText = 'display:none;width:37px;height:37px;flex-shrink:0;justify-content:center;align-items:center;background:transparent;cursor:pointer;border:none;padding:0;';
    btn.innerHTML = `<svg width="17" height="17" viewBox="0 0 24 24" fill="#111">...</svg>`;
    btn.addEventListener('click', (e) => {
        e.stopPropagation(); e.preventDefault();
        const block =
            document.querySelector<HTMLElement>('.icon-active')?.closest<HTMLElement>('[data-component-id^="my-block"]') ??
            document.querySelector<HTMLElement>('.elm-active')?.closest<HTMLElement>('[data-component-id^="my-block"]');
        if (block) setMyBlock(block);
    });
    linkTool.appendChild(btn);
}
```

`updateLinkToolBtnVisibility`에 가시성 로직 추가:

```ts
const myBtn = document.querySelector<HTMLElement>(`#divLinkTool .${SPW_MY_BTN_CLASS}`);
if (myBtn) {
    const isIn = !!iconActive?.closest('[data-component-id^="my-block"]') || !!elmActive?.closest('[data-component-id^="my-block"]');
    myBtn.style.display = isIn ? 'flex' : 'none';
}
```

### 4. React state 연결

```tsx
const [myBlock, setMyBlock] = useState<HTMLElement | null>(null);
// ...
{myBlock && <MyBlockEditor blockEl={myBlock} onClose={() => setMyBlock(null)} />}
```

## 버튼 스타일 규칙 (`#divLinkTool` 안 커스텀 버튼)

| 속성 | 값 |
|---|---|
| width / height | `37px` |
| background | `transparent` |
| border | `none` |
| SVG fill | `#111`, size `17×17` |
| display 초기값 | `none` (가시성 로직에서 `flex`로 전환) |

## 자주 하는 실수

**① 저장된 페이지 HTML 미업데이트**
마이그레이션 스크립트는 `SPW_CMS_COMPONENT`(컴포넌트 템플릿)만 업데이트한다.
이미 저장된 페이지(`SPW_CMS_PAGE`, `public/uploads/pages/*.html`)는 별도로 업데이트해야 한다.
→ 기존 블록을 삭제 후 재추가하거나 파일을 직접 수정한다.

**② `#divLinkTool`이 열리지 않음**
블록 안에 `<a>` 태그가 없거나, 저장된 페이지 HTML이 옛날 버전(`<div>` 사용)인 경우 발생한다.

**③ ContentBuilder Runtime 플러그인 중복 등록**
순수 HTML로 변환 완료된 컴포넌트를 `EditClient.tsx`의 `plugins` 배열에 계속 등록하면,
`reinitialize()` 시 재마운트되어 인라인 편집이 차단된다. **변환 완료 컴포넌트는 반드시 제거한다.**
