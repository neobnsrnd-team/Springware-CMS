## 🤖 Playwright 자동화 현황

### 공통 헬퍼 (`e2e/helpers/component-checks.ts`)

모든 컴포넌트 스펙 파일에서 `import` 하여 재사용하는 공통 함수 모음.

| 함수 | 검증 항목 | 적용 대상 |
|---|---|---|
| `runCommonChecks()` | 아래 전체를 한 번에 실행 | 모든 컴포넌트 |
| `checkNoHorizontalScroll()` | 가로 스크롤 발생 여부 | 모든 컴포넌트 |
| `checkNotOutsideViewport()` | 뷰포트 이탈 여부 | 모든 컴포넌트 |
| `checkMinFontSize()` | 폰트 크기 최소 12px | 모든 컴포넌트 |
| `checkMinTouchTarget()` | 터치 영역 최소 44px | 버튼·링크 있는 컴포넌트 |
| `checkImagesHaveAlt()` | 이미지 alt 속성 존재 | 이미지 포함 컴포넌트 |
| `checkKeyboardFocusable()` | 키보드 Tab 포커스 이동 | 인터랙티브 컴포넌트 |
| `checkComponentIdExists()` | `data-component-id` 속성 존재 | 순수 HTML 컴포넌트 전체 |

### 실행 방법

```bash
# 전체 뷰포트(mobile/web/responsive)에서 실행
npm run test:e2e

# 특정 컴포넌트 스펙만 실행
npm run test:e2e -- e2e/components/app-header.spec.ts

# 모바일 뷰포트만
npm run test:e2e -- --project=mobile

# UI 모드 (시각적 확인)
npm run test:e2e:ui
````

### 컴포넌트별 스펙 파일 현황

| 컴포넌트 | 스펙 파일 | 상태 |
|---|---|---|
| `app-header` | `e2e/components/app-header.spec.ts` | ✅ 작성 완료 |
| `info-card-slide` | `e2e/components/info-card-slide-editor.spec.ts` | ✅ 에디터 E2E 작성 완료 |
| `product-menu` ~ `popup-banner` | `e2e/components/*.spec.ts` | 🔲 미작성 |

> 신규 컴포넌트 스펙 추가 시: `runCommonChecks()` 한 줄로 공통 7개 항목 자동 처리, 이후 컴포넌트 고유 체크만 추가
